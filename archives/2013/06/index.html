游戏化的O2O开心农场应用狂想

寫過叁篇文章都在有啊無的瞎扯淡，不過從這篇開始就準備扯些有意思的東西了。

上周末從京都千里迢迢跑到西京去參加兩位大學同學共同的婚禮，期間有壹基友陪著跑東跑西，同時我注意到他只要閑下來就不停地玩壹款農場應用。要麼用iPhone，要麼用iPad，並且還因為誤操作損失了金幣而怒吼。於是我就在想，類似開心農場這樣的遊戲，鼎盛時期應該是在前幾年SNS 網路興起的那段時間吧，為何現在轉到移動端還是這樣讓人沉迷？在婚禮頭天晚上陪著新郎新娘預演時，另壹哥們開玩笑說：要是這個遊戲能加上實時天氣，肯定更好玩。

說者無意，聽者有心。作為在互聯網創業圈浸淫已久的屌絲從業者，馬上鎖定這點訊息開始深入思考。除了加入實時天氣，是否能夠做得更多呢？

不忘初心，方得始终

得益於開源的力量，Github + Octopress + 主題模版，這些已經足夠支撐起基本的博客功能。但是在慣性思維的影響下，我又開始琢磨是不是應該在這個基礎上再加上些什麼。想來想去，也沒想到放什麼合適，倒是想起了此前搞網站的經歷。

此前，由於工作的原因對Discuz 系統有過比較深入的接觸，所以就想自己用它來搭個網站，以償壹直以來擁有自己網站的小願望。只是做個網站框架容易，要想清楚做什麼內容、內容以什麼形態呈現，卻都不是容易事。於是自己開始了各種摸石頭。

I'm OK

在Windows 7 系統中部署Ruby 的開發環境確實讓人感覺有些蛋疼。第壹篇文章發布後，發現把Github 誤拼寫為“Gighub”，第貳天想立即改，結果本地怎麼也編譯不了。折騰來折騰去沒找到原因，還試圖通過虛擬機的方式來解決，還是無果。

今天參加大學哥們婚禮回來，首先想到的是這個問題還沒有解決，又突然靈感爆發，想起似乎是Windows 中環境變量參數設置的問題。於是對中文的支持問題就這樣解決了。

Hello Github

壹直想搭建壹個比較私人的博客，讓自己天馬行空的想法和日常的絮絮叨叨都能有地方安放。雖說用Wordpress 搭網站已經輕車熟路，只用花很少的時間就能做好，不過考慮到後續要花不少時間來張羅服務器相關的事，所以始終沒動。正好知道越來越多的朋友在Github 上搭建自己的博客，作為久不寫代碼、更不懂Ruby 的偽Geek，這種方式仍然是我的優選。盡管花了壹些時間來摸索，但最終還是把這個博客搭起來了，確實很值得、很有成就感的說。

感謝Github及Octopress提供的各種資源和服務！
